'docker篇'

发表于 2018-12-26 更新于 2019-09-15
前几天折腾docker在这里总结一下
  1. docker是什么？
    我认为docker算是一个载体，在他里面可以隔离运行一些应用。

  2. docker的容器和镜像。
    当时以为docker的容器和镜像是一个东西，docker其实不一样，当时就吃了大亏，你可以理解为docker的容器是镜像的一个实例，容器是动态变化的，镜像是死的，所以，你每次运行完镜像，觉得合适就可以保存容器，或者说更新镜像。
    更新镜像的命令：

    docker commit -m=”更新的信息” -a=”作者” 容器号 新名字

  3. docker的好处
    可以实现环境的快速部署，数据迁移做好铺垫，如果封装好一个镜像，就不会为环境搭建浪费事件。

  4. 什么是Dockerfile
    我刚开始以为Dockerfile 是一个镜像必须的文件，其实不是， 这个文件可以快速创建一个镜像。还没有深入docker 待补
    1. 最后附上docker的一些命令
      • docker run //运行一个镜像

        -d: 后台运行容器，并返回容器ID；

        -i: 以交互模式运行容器，通常与 -t 同时使用；

        -p: 端口映射，格式为：主机(宿主)端口:容器端口

        -t: 为容器重新分配一个伪输入终端，通常与 -i 同时使用；

        –name=”nginx-lb”: 为容器指定一个名称；

        –dns 8.8.8.8: 指定容器使用的DNS服务器，默认和宿主一致；

        –dns-search example.com: 指定容器DNS搜索域名，默认和宿主一致；

        -h “mars”: 指定容器的hostname；

        -e username=”ritchie”: 设置环境变量；

        –env-file=[]: 从指定文件读入环境变量；

        –cpuset=”0-2” or –cpuset=”0,1,2”: 绑定容器到指定CPU运行；

        -m :设置容器使用内存最大值；

        –net=”bridge”: 指定容器的网络连接类型，支持 bridge/host/none/container: 四种类型；

        –link=[]: 添加链接到另一个容器；

        –expose=[]: 开放一个端口或一组端口

算了命令有点有参照菜鸟教程吧

'linux动态调试工具gdb使用'

发表于 2018-12-20 更新于 2020-05-24
这里介绍一下linux下gdb的用法

调试程序

以当前有一个re1程序为例

  • gdb ./re1

设置断点，adr表示16进制地址，func表示函数名

  • b *0xadr
  • b func

删除断点，num为断点的编号。

  • d num

单步调试,发现直接按回车也可以

  • n

显示寄存器的内容以及寄存器加一个地址后内容,r为寄存器

  • x/nuf ($r+0xadr)
    其中
    • 是一个正整数，表示显示内存的长度，也就是说从当前地址向后显示几个地址（units）的内容。
  • 表示显示的格式（format），见下面“GDB打印数据显示格式”。如果地址所指的是null-terminated string，那么格式可以是s，如果地址是machine instruction，那么格式可以是i。默认初始使用十六进制格式。
  • 表示（the unit size）从当前地址往后请求的位宽大小。如果不指定的话，GDB默认是4个bytes。u参数可以用下面的字符来代替，b表示单字节，h表示双字节，w表示四字 节，g表示八字节。当我们指定了位宽长度后，GDB会从指内存定的内存地址开始，读写指定位宽大小，并把其当作一个值取出来。
